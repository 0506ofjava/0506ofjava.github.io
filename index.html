JVM自动内存管理概述

首先声明本部分内容起步阶段借鉴于《深入理解Java虚拟机 JVM高级特性与最佳实践》一书，而后续通过百度谷歌等工具外加部分自己的理解进行内容扩充。本文开始阶段，对内存管理进行一个简要的概括，以起到提纲挈领之功效，也便于快速掌握基础内容。


内存区域

JVM运行内存区域，即包括虚拟机栈、本地方法栈、程序计数器、堆内存和方法区五大区域。其中前三种为线程私有，而后两种为线程共享。通常前三种会随线程终止而回收，故不必特意调整参数。由于后两者分配和回收都是动态的，故内存垃圾回收主要针对于线程共享的两种（堆内存和方法区）。以上五种内存区域对比如下表所示：

Spring之旅

简化JAVA开发

  • 基于POJP的轻量级和最小侵入性编程
  • 通过依赖注入和面向接口松耦合
  • 基于切面和惯例进行声明式编程
  • 通过切面和模板减少样板式代码

激发POPJ的潜能

基于Spring构建的应用中，它的类通常没有任何痕迹表明你使用了Spring。最坏的场景是，一个类或许会使用注解，但它依旧是POJO（简单老式JAVA对象).Spring的非侵入式编程，意味着这个类可以在Spring应用和非Spring应用中都可以发挥同样的作用

依赖注入

DI功能的实现

如果每个对象负责管理与自己相互协作的对象的引用，将会导致高度耦合和难以测试的代码。简单点讲，就是一旦使用new关键字，就不可避免地陷入紧密耦合之中。通过DI，对象的依赖关系将由系统中负责协调对象的第三方组件在创建对象的时候进行设定，对象无需自行创建或管理它们的依赖关系。


构造器注入：将依赖的接口作为构造器参数传入，而对象自身不创建对象和依赖关系。也就是，利用参数传入而不是new关键字创建。

XML装配:创建应用组件之间协作的行为通常称为装配，而xml装配就是很常见的一种方式。（基于JAVA的配置:利用@configuration和@bean注解进行配置，可作为xml的替代方案）

观察它如何工作

Spring通过应用上下文（Application Context)装载bean的定义并把它们组装起来，Spring应用上下文全权负责对象的创建和组装。Spring自带多种应用上下文的实现，它们之间主要的区别仅仅在于如何加载配置。由xml进行的配置，可以使用ClassPathXmlApplicationContext进行加载和获取对象的引用。

应用切面

DI能够让相互协作的软件组件保持松散耦合，而面向切面编程则允许你把遍布各处的功能分离出来形成可重用的组件。关注点的调用经常散布到各个模块之中，而这些关注点并不是模块的核心业务。而利用AOP，系统范围内的关注点覆盖在它们所影响的组件之上。

例如：诗人在骑士战斗前后进行歌颂，然而如果不是面向切面编程，那么骑士将需要去调用诗人歌颂方法去实现功能，这本身就是一件很奇怪的事情。下面通过xml去实现切面编程,采用的是AspectJ切点表达式：

JDBC基础写法及优化

JDBC基础写法

JDBC实际就是服务器与数据库之间进行数据交换的桥梁，那究竟怎么才能搭好这座桥呢？以下将从背景知识、基本步骤、项目实践和总结反思四个方面进行阐述。

背景知识

项目总体涉及到以下几个方面的基础知识

  • 反射技术：主要体现在驱动的加载过程，如Class.forName(“com.mysql.jdbc.Drive”)
  • MVC框架：MVC即模型（MODEL）-表示（VIEW）-控制（CONTROL），如jsp-servlet-javabean、structs2
  • Sql语句：增删改查（CRUD），其中查询为重中之重
  • 泛型：常与容器相关
  • 设计模式：
  • JavaDoc生成方法及优化：思考怎么生成优雅的JavaDoc文件，方便他人与自己的阅读
  • 集合类：ArrayList()、HashMap()的使用，包括声明、赋值、添加以及遍历

对于MVC的模型层，多讲两句。以当前项目的观察而言，模型层为主要的业务逻辑层。简单的包括：数据库映射（操作对象）、Dao层（数据访问层面向表，建议只针对某张表做原子化操作：增删改查）、Service层（对Dao层进行封装，面向业务：事务控制）

基本步骤

  • Dao层：面向表单，负责针对表单做原子化操作：增删改查
    1. 加载数据库驱动：Class.forName(“com.mysql.jdbc.Driver”)
    2. 创建连接指定数据库的对象：Connection connection=DriveManager.getConnection(URL，NAME，PASSWORD)
    3. 创建执行SQL语句的对象：Statement stament=connection.createStatement()、或者PreparedStatement preparedStatement=connection.preparedStatement(sql)
    4. 查询结果（查询）：ResultSet resultset=preparedStatement.executeQuery()、ResultSet resultSet=Statement.executeQuery(sql)
    5. 简单执行(增加、删除、更改)：preparedStatement.excute()
    6. 增删改查：增（add方法：INSERT INTO tablename (id,name,age) values(?,?,?);)、删（DELETE FROM tablename WHERE id=?;)、改（UPDATE tablename SET name=? WHERE id=?;)、查（SELECTE * FROM tablename WHERE id=?）
  • Service层：创建Dao的实例对象，调用Dao类方法

Java I/O流简要总结

Java的 I/O流主要内容有：字节流、字符流以及序列化和反序列化等内容，其是应用程序与硬盘等外设进行交互的桥梁。Java中也提供了丰富的API支持，读写等操作相对还是比较简单的。下表提供了一些API的总结，仅供参考。

字节流 FileInputStream(File file) read()
read(byte[] bytes)		 /
字节流 FileInputStream(File file) write()
write(byte[] bytes)		 /
字符流 InputStreamReader(InputStream in) read()
read(char[] char)		 /
字符流 OutputStreamReader(OutputStream out) write()
write(char[] char)		 write(chars,0,c);
write(chars,0,chars.length);
字符流过滤器 BufferedReader(Reader inputstreamreader) readLine() readLine()不需要手动换行
字符流过滤器 BufferedWriter(Writer outputstreamwriter) writerLine() writerLine()需要手动换行
字符流过滤器 PrintWrite(Writer outputstreamwriter) println() println()自动换行
序列化 ObjectInputStream() readObject() 重写方法，自行序列化
反序列化 ObjectOutputStream() writeObject() 重写方法，自行反序列化
