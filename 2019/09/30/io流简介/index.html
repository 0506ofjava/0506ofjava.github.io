Java I/O流简要总结

Java的 I/O流主要内容有：字节流、字符流以及序列化和反序列化等内容，其是应用程序与硬盘等外设进行交互的桥梁。Java中也提供了丰富的API支持，读写等操作相对还是比较简单的。下表提供了一些API的总结，仅供参考。

类别 API名称 常用方法 备注
字节流 FileInputStream(File file) read()
read(byte[] bytes)		 /
字节流 FileInputStream(File file) write()
write(byte[] bytes)		 /
字符流 InputStreamReader(InputStream in) read()
read(char[] char)		 /
字符流 OutputStreamReader(OutputStream out) write()
write(char[] char)		 write(chars,0,c);
write(chars,0,chars.length);
字符流过滤器 BufferedReader(Reader inputstreamreader) readLine() readLine()不需要手动换行
字符流过滤器 BufferedWriter(Writer outputstreamwriter) writerLine() writerLine()需要手动换行
字符流过滤器 PrintWrite(Writer outputstreamwriter) println() println()自动换行
序列化 ObjectInputStream() readObject() 重写方法，自行序列化
反序列化 ObjectOutputStream() writeObject() 重写方法，自行反序列化
  1. 由于BufferedWrite()需要手动换行，比较麻烦。所以一般使用PrintWrite代替它，进而与BufferedReader搭配使用。
  2. 序列化与反序列化涉及一个默认序列化的过程，这个是重点。
  3. 最后一个单词确认属于哪种流

字符流复制文件

当write参数里面为c时，可以正常读取（必须要执行flush（）方法，否则会出现读取异常。）

当write参数为chars.length时反而没办法将文件写完整，这个就显得异常奇怪。因为chars.length比c的值更大，然而却读取不完整。

读取结果：可以看到out 2（复制后的文件：9.7 Kb）虽然比out（原文件：7.4 Kb）的文件更占空间，但是内容却是不完整的。

总之，注意以下三点：

  1. 一定要执行flush()方法;
  2. 批量读取时，write内应该读取到c就停止;
  3. 指定对应的字符编码

序列化内容

序列化实际就是对象的输入输出，涉及到的API有ObjectInputStream()和ObjectOutputStream()，其下的重要方法为readObject()与writeObjete（）。当需要自己序列化时，需要重写readObject()与writeObjete（）。
