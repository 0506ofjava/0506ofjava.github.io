JDBC基础写法

JDBC实际就是服务器与数据库之间进行数据交换的桥梁，那究竟怎么才能搭好这座桥呢？以下将从背景知识、基本步骤、项目实践和总结反思四个方面进行阐述。

背景知识

项目总体涉及到以下几个方面的基础知识

  • 反射技术：主要体现在驱动的加载过程，如Class.forName(“com.mysql.jdbc.Drive”)
  • MVC框架：MVC即模型（MODEL）-表示（VIEW）-控制（CONTROL），如jsp-servlet-javabean、structs2
  • Sql语句：增删改查（CRUD），其中查询为重中之重
  • 泛型：常与容器相关
  • 设计模式：
  • JavaDoc生成方法及优化：思考怎么生成优雅的JavaDoc文件，方便他人与自己的阅读
  • 集合类：ArrayList()、HashMap()的使用，包括声明、赋值、添加以及遍历

对于MVC的模型层，多讲两句。以当前项目的观察而言，模型层为主要的业务逻辑层。简单的包括：数据库映射（操作对象）、Dao层（数据访问层面向表，建议只针对某张表做原子化操作：增删改查）、Service层（对Dao层进行封装，面向业务：事务控制）

基本步骤

  • Dao层：面向表单，负责针对表单做原子化操作：增删改查
    1. 加载数据库驱动：Class.forName(“com.mysql.jdbc.Driver”)
    2. 创建连接指定数据库的对象：Connection connection=DriveManager.getConnection(URL，NAME，PASSWORD)
    3. 创建执行SQL语句的对象：Statement stament=connection.createStatement()、或者PreparedStatement preparedStatement=connection.preparedStatement(sql)
    4. 查询结果（查询）：ResultSet resultset=preparedStatement.executeQuery()、ResultSet resultSet=Statement.executeQuery(sql)
    5. 简单执行(增加、删除、更改)：preparedStatement.excute()
    6. 增删改查：增（add方法：INSERT INTO tablename (id,name,age) values(?,?,?);)、删（DELETE FROM tablename WHERE id=?;)、改（UPDATE tablename SET name=? WHERE id=?;)、查（SELECTE * FROM tablename WHERE id=?）
  • Service层：创建Dao的实例对象，调用Dao类方法
